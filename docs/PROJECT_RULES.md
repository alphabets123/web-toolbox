# 프로젝트 관리 규칙 (Project Rules)

이 문서는 Snap-Task 프로젝트의 일관된 유지보수를 위한 규칙을 정의합니다. 모든 AI 에이전트 및 개발자는 이 규칙을 준수해야 합니다.

## 1. 에이전트 버전 관리 (youtube.js)

- **형식**: `v{YYYYMMDD}-{SERIAL}`
- **설명**: 
    - `YYYYMMDD`: 수정한 날짜 (예: 20260327)
    - `SERIAL`: 해당 날짜의 수정 횟수 (3자리 숫자로 표기 권장, 예: 014)
- **적용**: `youtube.js` 상단의 `VERSION` 상수를 매수정 시마다 업데이트합니다.
- **예시**: `const VERSION = 'v20260327-014';`

## 2. 코드 및 계획서 언어

- 모든 계획서, 보고서, 워크스루 및 사용자 안내는 **한글**로 작성하는 것을 원칙으로 합니다.
- 코드 주석 및 로그 메시지도 가급적 한글을 사용하여 사용자 친화적으로 유지합니다.
