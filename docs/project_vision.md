# Web Toolbox: 프로젝트 비전 및 장단점 분석

본 문서는 "정보 유출 없는 빠르고 단순한 웹 브라우저 기반 생산성 툴"이라는 모토 아래 기획된 전체 시스템의 특징과 장단점을 요약한 것입니다.

## 🌟 전체 시스템의 핵심 특징 (모토 기반)
모든 데이터(영상, 이미지)가 인터넷을 통해 서버로 전송되지 않고, **사용자의 브라우저(컴퓨터/스마트폰 자체 성능) 안에서만 처리**됩니다. 
이를 위해 `WebAssembly (FFmpeg.wasm)`나 `Web AI (ONNX Runtime Web)` 같은 최신 브라우저 기술을 사용하게 됩니다.

## 👍 막강한 장점 (Pros)
1. **완벽한 정보 보안 (Privacy):** 사용자 파일이 외부 서버로 1바이트도 전송되지 않으므로, 회사 기밀 문서나 개인적인 사진/영상을 처리할 때 유출 우려가 전혀 없습니다. (가장 큰 차별점입니다!)
2. **서버 유지비 "0원" (Zero Cost):** 영상 처리나 AI 작업은 서버 비용이 엄청나게 많이 들지만, 이 방식은 사용자의 기기 성능을 빌려 쓰기 때문에 트래픽 비용(클라우드플레어 Pages 무료 플랜 가능) 외에 서버 유지비가 들지 않습니다.
3. **무설치 및 플랫폼 독립성:** 윈도우, 맥, 심지어 모바일 환경에서도 브라우저만 켜면 언제 어디서든 접속해서 즉시 사용할 수 있습니다. 프로그램 설치의 번거로움이 없습니다.
4. **빠른 작업 속도:** 대용량 동영상을 편집할 때 수십~수백 MB의 파일을 답답하게 서버에 업로드/다운로드할 필요 없이, 로컬 스토리지에서 즉시 불러와서 저장합니다.

## 👎 고려해야 할 단점 및 기술적 한계 (Cons)
1. **사용자 기기 성능에 의존:** 서버가 아닌 방문자 본인의 스마트폰/PC CPU 자원을 100% 사용하기 때문에, 저사양 기기나 구형 스마폰에서는 렌더링(합치기)이나 배경 제거 속도가 느릴 수 있습니다.
2. **초기 로딩 속도:** AI 기능을 브라우저에서 돌리기 위해, 웹사이트 접속 시 인공지능 모델(Model)이나 WebAssembly 엔진(수십 MB)을 처음 한 번 다운로드해야 합니다. (다만 브라우저 캐싱을 통해 두 번째 접속부터는 빠릅니다.)
3. **일부 모바일 브라우저 호환성:** 너무 최신 기술(특히 멀티스레드와 관련된 SharedArrayBuffer 등)을 요구하기 때문에, 구형 브라우저나 일부 기기 환경에서는 까다로운 설정이 필요할 수 있습니다.

---

## 🚀 코어 기능별 기술 요약평

### 1. 유튜브 영상 다운로드
- 사실 이 부분이 가장 까다롭습니다. 유튜브의 철저한 방어벽(CORS 에러 등) 때문에 순수 브라우저만으로는 해결하기 어렵고, 주소를 우회시켜주는 아주 가벼운 '프록시 형태의 중계 API(예: Cloudflare Workers 활용)'가 보조적으로 필요할 수 있습니다.

### 2. 동영상 구간 컷 + 합치기 (FFmpeg.wasm)
- **가장 수요가 많으면서도 기술적 난이도가 시도해볼 만한 영역입니다.** 단순 컷편집은 서버 업로드 없이 브라우저에서도 상당히 훌륭한 속도와 퍼포먼스를 보여줍니다.

### 3. 이미지 배경 제거 (ONNX Web / MediaPipe)
- 최근 웹 AI 발전으로 브라우저 내에서 정말 깜짝 놀랄 수준의 누끼(배경 제거) 품질을 서버 전송 없이 얻을 수 있습니다.

---
**💡 향후 모토 및 수익화:** "서버 비용 걱정 없이 트래픽/광고 수익을 노리면서, 압도적인 보안성을 제공하는 툴"은 현재 최신 웹 생태계의 발전 방향과 가장 완벽하게 일치하는 현실적인 비즈니스 모델입니다.
